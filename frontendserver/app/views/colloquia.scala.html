Web Interface for Amandroid


Presenter: Rohit Sharma


Advisor: Sankardas Roy


Android is a very popular platform with millions of users. However, malicious or vulnerable apps have been developed, which causes security concerns. Amandroid is a static analysis framework for vetting android apps. Amandroid has gained a fair bit of attention from the research community, but it does not yet support a real-time vet request from users. So, a web service for amandroid will enable an online vetting facility. In other words, with amandroid web service all users can analyze and test android apps and get the analysis results.

Amandroid is built on the Akka framework, which includes a runtime toolkit to design and develop concurrent, distributed and message-driven application based on the Actor Model. Web service for amandroid is built on the Play-framework that is built on Akka and uses a highly scalable web-friendly architecture. In this talk, I will explain the amandroid web service and its need. I will then present the current architecture of this web service. I will explain the challenges we faced in developing this web architecture and how we have come to the solution. I will give a demo of the current application. I will conclude the presentation by explaining the future enhancement for the web service.

Brief Bio: Rohit Sharma earned a Bachelor of Engineering in computer science from SVITS Indore, India in 2011. He is a MS candidate at Bowling Green State University where he works with Dr. Sankardas Roy. Prior to Bowling Green State University, he worked in industry for 3 years as a software engineer. His research interests are in Web Development and Programming Languages.
